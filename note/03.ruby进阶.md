### 类,对象,模块

#### 变量

- local 变量
  - ruby的local变量没有变量提升一说, 函数内变量与外部变量完全隔离, 互不影响

```ruby
def basic_method
  x = 50
  puts x
end

x = 10
basic_method # 50
puts x # 10
```
- global 变量
  - ruby中也有全局变量, 是在变量前加上$符号
  - 不推荐在ruby这种纯OOP的语言中使用全局变量

```ruby
$global_x = 30

def second_method
  puts $global_x
end

second_method # 30
```

- 实例变量
  - 用`@`符号表示的变量是实例变量
  - 相当于其他语言的this

```ruby
class Square
  def initialize(side_length)
    @side_length = side_length
  end

  def area
    @side_length * @side_length
  end
end
```

- 类变量
  - 以 `@@` 开始的变量即是类变量
  - 相当于Java里的static, 直接挂在类上面的,但是区别在于不用显示声明出来就能用.
  - 尽量减少使用类变量以避免耦合的出现.

```ruby
class Square
  def initialize
    if defined?(@@number_of_squares)
      @@number_of_squares += 1
    else
      @@number_of_squares = 1
    end
  end

  # 
  def self.count
    @@number_of_squares
  end
end

a = Square.new
b = Square.new

puts Square.count # 2
```

#### 继承

- 和传统OOP一样, 只能单继承关系
- 继承的子类会覆盖父类的同名方法和属性
- super关键字会调用父类的同名方法

```ruby
class Parent
  def method1
    puts "ok, comes from parent method1"
  end

  def method2
    puts "ok, comes from parent method2"
  end
end

class Child < Parent
  def method2
    puts "ok, comes from child method2"
  end
end

# ok, comes from parent method1
puts Child.new.method1
# ok, comes from child method2
puts Child.new.method2
```

```ruby
class Person
  def initialize(name)
    @name = name
  end

  def name
    @name
  end
end

class Teacher < Person
  # super会直接拿到父类中同名方法并执行
  # 所以这里的super返回了父类中的@name的结果, 拼接的结果就是Dr. stan
  def name
    "Dr. " + super
  end
end

puts Teacher.new("stan").name
```

#### 复写(Override)

- 存在于Ruby中的模块能被轻易的Override, 比如String的length方法, 只要声明的类名一样,就能Override
- 基于这个特性, Ruby的插件能轻易的增强一些通用类
- 同样, 如果是应用的话, 多次声明同一个类只会修改相同的方法,其他的内容不会丢失.

```ruby
# 3. example3
class String
  def length
    20
  end
end

# 20
puts "abc".length
```

#### 反射(Reflection)

- 和java的反射概念类似, 能在ruby解释器运行的时候拿到对应类所有的方法,属性等.
- 作用是进行动态编译.
  
```ruby
# 1. methods 反射方法, 直接拿到实例所有能用的方法, 本身有的+继承的
a = "This is a test"
puts a.methods.join(' ')

# 2. instance_variables, 拿到实例所有能用的成员变量名
class Person1
  attr_accessor :name, :age
end

p = Person1.new
p.name = "stan"
p.age = 10
# @name
# @age
puts p.instance_variables
```

#### 封装(encapsulation)

- 即类的访问权限管理, ruby中分为以下几个
- public
  - 默认,都能访问,不用刻意写
- private
  - 仅允许类内部调用
- protected
  - 仅允许类内部与继承相关类使用
- 以上关键字能单独列出来用,声明一堆方法是private的,也能单独声明某个方法是属性某种权限的(此种情况下声明要使用Symbol)

```ruby
# 1. public protected private
class Person2
  def initialize(name)
    set_name(name)
  end

  def name
    @first_name + " " + @last_name
  end

  # 以下所有方法都是public的
  public 

  def say_hi
    puts "Hi, I'm #{name}"
  end

  # 以下所有方法都是protected的
  protected

  def say_yes
    puts "Yes"
  end
  
  # 以下所有方法都是private的
  private
  
  def set_name(name)
    # 和python一样, 允许多值接收
    first_name, last_name = name.split(/\s+/)
    set_first_name(first_name)
    set_last_name(last_name)
  end

  def set_first_name(name)
    @first_name = name
  end

  def set_last_name(name)
    @last_name = name
  end
end

p2 = Person2.new("stan yang")
# stan yang
puts p2.name
# Hi, I'm stan yang
p2.say_hi
# error, only supported within inherited classes
p2.say_yes


# 2. 个别方法写一行private,protected
class Person3
  def initialize(age)
    @age = age 
  end

  def secret_method_1
    puts 1
  end

  def secret_method_2
    puts 2
  end

  def secret_method_3
    puts 3
  end

  def age
    @age
  end

  # 必须得这样写, 因为是从上往下读的, 不能先函数定义进行设定
  # 一行形式写法必须用symbol定义, 只能是方法
  private :secret_method_1, :secret_method_2
  protected :secret_method_3, :age
end

# 这种写法VSCODE能.出方法名出来, 但是调用不会成功
Person3.new.secret_method_3
```

#### 多态

- 和Java的概念一样, 子类能根据父类有自己的特性
- 比如不同数据类型调用.to_s方法, 也属于多态.

```ruby
# 1.
class Animal
  # 相当于getter
  attr_accessor :name

  def initialize(name)
    @name = name
  end
end

class Cat < Animal
  def talk
    "aaa"
  end
end

class Dog < Animal
  def talk
    "bbb"
  end
end

[Cat.new("ZX"), Dog.new("TA")].each { |animal| puts animal.talk}

# 2. 这个也是多态, 不同的Class相同的方法得出不同的结果
puts 1000.to_s
puts [1,2,3].to_s
puts ({ name: 'Fred', age:10 }).to_s
```

#### 类嵌套

- 即类中有类的模式,用于类的分组管理
- 外部访问形式是通过::的形式
- 内部能直接访问

```ruby
# 1. 
class Drawing
  class Line
  end

  class Circle
    def say_circle
      puts "circle here"
    end
  end

  def self.get_Circle
    Circle.new
  end
end

circle1 = Drawing.get_Circle
# circle here
circle1.say_circle

circle2 = Drawing::Circle.new
# circle here
circle2.say_circle
```

#### 常量作用域

- Ruby中的常量存在全局作用域, 当某个类中有同名常量时, 作用域之间相互隔离
- 要访问到某个类下面的常量时,也是用::来获取
- Recap: Ruby中的常量用首字母大写标记

```ruby
# 1.
Price = 100
class Home
  Price = 85
  def self.get_total_price
    3 * Price
  end
end

puts Price # 100
puts Home::Price # 85
puts Home.get_total_price # 255
```

#### 模块, 名称空间, 混入(mixins)

- Module 用于收集 Ruby当中的类, 并将他们收纳进一个独立且隔离的空间里
- 被Module收纳的类名,常量,方法与外部完全不相干, 会形成自己的名称空间
- Module 命名使用的是`大驼峰`的方式
- Module 通过 `require` 关键字导入
- 拿到Module下某个类的方式也是::

```ruby
# 1. 
module ToolBox
  class Ruler
    attr_accessor :length
  end
end

module Country
  class Ruler
    attr_accessor :name
  end
end

a = ToolBox::Ruler.new
a.length = 50
b = Country::Ruler.new
b.name = "stan"

puts a.length # 50
puts b.name # stan
```

- 混入(mixins)的概念和Vue的mixin一样, 能随意将module混入到任意一个类里
- 最关键的是mixins是没有限制的,能混入无限多.
- 在功能性函数, 常量等分开管理的情况下非常有用
- 使用关键字 `include` 对module 进行混入操作. 这里有几个注意点.
  - 1.class里能进行`include`操作, 非class里也能进行混入操作.
  - 2. 混入后能拿到module里所有的内容, 相当于成为某个类/作用域的一部分了, 不需要再加模块名称进行调用.

```ruby
# 1. class混入
module HelperFeature
  def class_name
    self.class.to_s
  end
  def say_hi
    puts "hi"
  end
end

class Game
  include HelperFeature
end

x = Game.new
puts x.class_name # Game

# 2. Scope混入
include HelperFeature
say_hi # hi
```

- 两个有用的Module
  - 1. Enumerable
    - 提供了统计和迭代相关的方法
    - 比如: collect, find, include? , select, sort ...
    - 需要类中Override `each` 方法进行触发
    - 所有Array都能使用(只要能调用到each,each里放yield预留出代码块, 就能通过include Enumerable模块来用各种遍历操作进行数组相关的处理)

```ruby
# 1. Enumerable
# collect: 返回一个加工后的Array, 相当于JS里的every
puts [1,2,3,4].collect {|i| i.to_s + "A" }

# detect: 返回符合条件的第一个元素, 相当于JS里的find
puts [1,2,3,4].detect {|i| i.between?(2,3)}

# select: 返回符合条件的元素数组, 相当于JS里的filter
puts [1,2,3,4].select {|i| i.between?(2,3)}

# sort: 返回数组升序排序后的数组, 相当于JS里的sort
puts [3,4,1,2].sort

# max: 返回数组里最大的元素
puts [1,2,3,4].max

# min: 返回数组里最小的元素
puts [1,2,3,4].min

# 2. 类中使用Enumerable模块
# 自定义each
class AllVowels
  include Enumerable
  VOWELS = %w{a e i o u}

  # 自定义了each, yield 会生成code block, 这块位置会挖空, 等其他遍历方法传入进来使用这里的变量
  def each
    VOWELS.each {|vowel| yield vowel}
  end
end

x = AllVowels.new

# 这里的{|i| i + "x"}, 丢到了上面yield的位置替换它的内容
x.collect {|i| i + "x"}

# ["ax", "ex", "ix", "ox", "ux"] 
puts x

# "o"
x.detect {|i| i > "j"}

# ["o", "u"] 
x.select { |i| i > "j" }

# ["a", "e", "i", "o", "u"]
x.sort

# "u"
x.max

# "a" 
x.min
```

  - 2. Comparable
    - 提供了比较相关的方法
    - 比如: between?,
    - 需要在类中使用 `<=>` 来进行触发
    - 这样就能通过类的某些属性来自定义如何去比较实例了.
    - Recap: 
      - 1 <=> 2 返回 -1
      - 1 <=> 1 返回 0
      - 2 <=> 1 返回 1
      - 判断逻辑: 左边是否大于右边
```ruby
# 1. 
class Song
  include Comparable

  attr_accessor :length

  def <=>(other)
    @length <=> other.length
  end

  def initialize(song_name, length)
    @song_name = song_name
    @length = length
  end
 end

a = Song.new('aaa', 143)
b = Song.new('bbb', 544)
c = Song.new('ccc', 60)

puts a > b # false
puts b >= c # true
puts a.between?(c,b) # true
```

#### 类型检查(3.0新增特性)

- 从3.0版本开始, ruby提供了类型检查的机制, 叫做RBS

- 启用RBS, 需要一个额外的与.rb文件同名的.rbs文件
- rbs的文件结构基本和ruby一模一样,只是设定了参数类型和返回类型(感觉挺臃肿的...)
- `https://github.com/ruby/rbs.` ruby3.0新增的东西, 刚开始搞.

#### 快速创建Data对象的Struct类

- ruby和java一样, 都是完全的面向对象语言, 那也就意味着, 对象类型数据的存储也都必须有对应的类来实例化接收.
- 为了避免这种反复创建为接收数据而存在的代码, ruby提供了Struct类来进行类的快速构造.

```ruby
# 1. Struct类快速构建一个Data类
#   这种形式构建的类,参数都是可选的,并不是强制的
Person = Struct.new(:name, :age, :gender)

my_friend= Person.new("zhangsan", 16, "male")

puts my_friend.gender

# 相比较下面这种定义方式, 省略掉的代码不是一点半点, 但是这种参数不能省略,attr_accessor修饰存在, 参数必须都传.
class Person
  attr_accessor :name, :gender, :age
  def initialize(name, gender, age)
    @name = name
    @gender = gender
    @age = age
  end 
end
```

#### attr_accessor 详解

- 是getter和setter的结合体
- 用了就省掉了刻意定义函数赋值的内容了.

```ruby
# 1.
class Person
     attr_accessor :name, :age
end
# 等价于
class Person
  def name=(name)
    @name = name
  end

  def age=(age)
    @age = age
  end
end
# 可以看到是getter和setter的优化写法
# def 那里定义不带=号,则只是setter.
```
